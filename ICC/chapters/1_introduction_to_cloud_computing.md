## 1.1 Evolution of Cloud Computing ‚≠ê‚≠ê‚≠ê

### Historical Timeline

| Era                   | Period      | Key Developments                    |
| --------------------- | ----------- | ----------------------------------- |
| **Mainframe**         | 1960s       | Time-sharing, centralized computing |
| **Client-Server**     | 1980s       | Distributed processing, LANs        |
| **Internet**          | 1990s       | Web applications, e-commerce        |
| **Grid Computing**    | Early 2000s | Distributed resource sharing        |
| **Utility Computing** | Early 2000s | Pay-per-use model                   |
| **Cloud Computing**   | 2006+       | Virtualized, on-demand services     |

### Key Milestones

- **1969:** ARPANET (Internet precursor)
- **1999:** Salesforce.com (SaaS pioneer)
- **2002:** Amazon Web Services launched
- **2006:** AWS EC2 and S3 released
- **2008:** Google App Engine
- **2010:** Microsoft Azure
- **2013:** Docker (containerization)

### Enabling Technologies

1. **Virtualization:** Multiple VMs on single hardware
2. **High-speed Networks:** Broadband, fiber
3. **Web 2.0:** Interactive web applications
4. **Service-Oriented Architecture (SOA)**
5. **Utility Computing:** Pay-per-use model

## 1.2 Cloud Computing Definition ‚≠ê‚≠ê‚≠ê

### NIST Definition

"Cloud computing is a model for enabling ubiquitous, convenient, on-demand network access to a shared pool of configurable computing resources (e.g., networks, servers, storage, applications, and services) that can be rapidly provisioned and released with minimal management effort or service provider interaction."

### Simple Definition

**Cloud Computing:** Delivery of computing services (servers, storage, databases, networking, software) over the internet ("the cloud") with pay-as-you-go pricing.

---

## 1.3 Cloud Characteristics (NIST Essential Characteristics) ‚≠ê‚≠ê‚≠ê

### 1. On-Demand Self-Service

- Users provision resources without human interaction
- Web portal or API access
- Instant deployment

### 2. Broad Network Access

- Available over network from any device
- Standard protocols (HTTP, HTTPS)
- Cross-platform accessibility

### 3. Resource Pooling

- Multi-tenant model
- Resources dynamically assigned
- Location independence

### 4. Rapid Elasticity

- Scale up/down quickly
- Appears unlimited to users
- Automatic scaling possible

### 5. Measured Service

- Resource usage monitored
- Pay-per-use billing
- Transparency for users and providers

## 1.5 Benefits of Cloud Computing ‚≠ê‚≠ê‚≠ê

### 1. Cost Efficiency

- No upfront capital investment
- Pay only for what you use
- Reduced IT staff requirements
- Lower operational costs

### 2. Scalability and Elasticity

- Scale up during peak demand
- Scale down when not needed
- Handle unpredictable workloads

### 3. Accessibility

- Access from anywhere with internet
- Support mobile and remote work
- Collaboration enabled

### 4. Reliability and Availability

- Built-in redundancy
- High uptime SLAs (99.9%+)
- Data backup included

### 5. Speed and Agility

- Quick deployment (minutes vs weeks)
- Faster time-to-market
- Easy experimentation

### 6. Automatic Updates

- Provider handles patches
- Always latest versions
- Security updates applied

### 7. Environmental Benefits

- Better resource utilization
- Reduced energy consumption
- Smaller carbon footprint

## 1.6 Challenges of Cloud Computing ‚≠ê‚≠ê

### 1. Security Concerns

- Data in third-party control
- Multi-tenancy risks
- Compliance challenges

### 2. Vendor Lock-in

- Difficulty switching providers
- Proprietary APIs
- Data migration challenges

### 3. Limited Control

- Less customization
- Dependency on provider
- Limited visibility

### 4. Network Dependency

- Requires internet connectivity
- Latency issues
- Bandwidth limitations

### 5. Compliance Issues

- Data sovereignty
- Industry regulations
- Audit requirements
